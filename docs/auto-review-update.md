# Auto Review Update

## Trigger Phrases

当用户说下面这类话时，优先按自然语言工作流处理：

```text
今天有什么要复习的
今天的复习提醒
复习完了
今天的错题复习完了
物理复习完了
数学复习完了，掌握得一般
```

## Intent Routing

### A. 查询待复习内容

适用于：

- `今天有什么要复习的`
- `今天的复习提醒`

处理步骤：

1. 查询今日待复习内容
2. 按学科分组
3. 只返回文字列表
4. 提示用户下一步可要求发送某科或全部复习 PDF

### B. 更新复习进度

适用于：

- `复习完了`
- `今天的错题复习完了`
- `物理复习完了`

处理步骤：

1. 识别为复习更新意图
2. 从上下文推断学生、学科、掌握情况
3. 调用 `update-review.py`
4. 返回更新数量、学科统计、下次复习日期

## Default Command

```bash
python3 skills/mistake-notebook/scripts/update-review.py \
  --student <学生姓名> \
  --today \
  --mastered good
```

## Response Templates

### 查询待复习内容

```text
📅 今日复习提醒

学生：<姓名>
待复习：<数量> 道

<按学科分组的条目>

可继续：
- 发送数学的复习内容
- 发送物理的复习内容
- 发送所有复习内容
```

### 更新成功

```text
✅ 复习进度已更新！

今日完成复习：<总数> 道
<学科统计>

下次复习日期：<日期>
```

### 无待复习内容

```text
🎉 今天没有待复习的错题。
```

## Best Practices

- 用户查询复习内容时，先给文字列表，不直接导出文件
- 用户说“复习完了”时，默认优先批量更新
- 掌握情况不明确时，可先按 `good` 处理，必要时补问
- 回复保持亲和、直接、专业
